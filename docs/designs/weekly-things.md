## Vision

Weekly summary becomes the primary family-sharing artifact. Users speak all week;
on Monday they tap one button and a warm, dialect-friendly paper is sent to family
chat. The weekly becomes the "family letter" — not a dashboard, not a report —
just a clear, warm note that says "here's what grandma was up to this week."

## Technical Contracts

### Cache lifecycle
- `GET /weekly/{week_start}`: returns `WeeklySummaryResponse` with `stale: false, regen_count: N` (fresh), `stale: true, regen_count: N` (stale), or 404 (never generated)
- `POST /weekly`: cache-first; if stale or none, generates and saves; returns `stale: false, regen_count: N`
- `POST /weekly` with `force=True`: counts ALL AuditResults for user×week (stale+fresh); if ≥5 → 429 + `"这个礼拜已经理了好几次了，下次再来。"`; otherwise inserts fresh record (stale records kept as history, NOT deleted)
- Stale marking: `is_stale=True` on AuditResult when entry in same week is created/updated/dismissed
- `regen_count`: always `COUNT(*) WHERE user_id=X AND week_start=Y AND audit_type="miniapp_weekly"` (stale+fresh)

### LLM opening paragraph
- Model: configured OpenAI-compatible provider via `get_chat_client()` + `chat_model()` (`deepseek-v3.2` default) — NOT Claude/Anthropic
- Extracted to `async def _generate_opening_sentence(labels: list[str], items: list[str]) -> str`
- Input: top 3 category labels + up to 5 display_text items (≤200 chars)
- Prompt: `"用温暖的沪语风格写一句话，总结这个礼拜用户主要做了：{labels}。不超过50个字。"`
- Temperature: 0.7 (warmer, more varied than categorization's 0.2)
- Timeout: 3s; on timeout or exception, return fallback text silently
- Example output: `"这个礼拜，你买菜做饭、照顾家人，还学了不少新东西——忙忙碌碌，但都是有意思的事。"`
- Fallback text (on timeout/error): `"这个礼拜，你讲过的话已经帮你整理好了。"`
- Rate limit: max 5 regenerations per user per week (count ALL AuditResult rows, stale+fresh); HTTP 429 + `"这个礼拜已经理了好几次了，下次再来。"` on force=True when count ≥5

### GET-before-POST flow
1. `GET /weekly/{week_start}` → if 200 with `stale: false`, show immediately
2. If 200 with `stale: true`, show summary + `重新理一理` button
3. If 404, call `POST /weekly` (body: `{week_start}`) to generate
4. `重新理一理` → `POST /weekly` (body: `{week_start, force: true}`) → refresh page data

## Design Decisions (from /plan-design-review 2026-04-19)

### `重新理一理` button spec
- Position: `.bottom-actions`, between `发给屋里人看看` (primary) and `再讲一段` (secondary)
- Condition: `wx:if="{{summary.stale}}"`
- Style: `.secondary-action` class — white bg, `--color-border` border, `--color-red-dark` text
- Loading state: `loading="true" disabled="true"` + copy `正在重新理` (button-level; rest of page stays visible)
- Disabled state (≥5 regens or 429): copy changes to `今礼拜理好几次了`; `disabled="true"`; uses `.secondary-action[disabled]` style

### Stale visual hint
- Element: `<text wx:if="{{summary.stale}}" class="muted">讲过新的以后，可以重新理一理</text>`
- Position: below `{{summary.date_range}}`, before the opening-note section
- Font: `34rpx`, `--color-muted` (#5F6673)
- Purpose: makes the regen button discoverable without scrolling to bottom

### LLM output sanitization

## Reviewer Concerns

1. **Rate limit counting** — count check must happen BEFORE insert in force=true flow
2. **LLM fallback template** — use `"这个礼拜，你讲过的话已经帮你整理好了。"` as generic fallback
3. **Regen button** — disable after 5th attempt (copy: `今礼拜理好几次了`), toast on 429, not just one or the other
